## 1. Objetivo

Detector de secciones legales mediante regex/heurísticas del foro judicial
chileno. Catálogo UNIFICADO de patrones (transversales a todos los
procedimientos civiles). Opera sobre texto ya normalizado (output de 7.07a).

Produce `DetectedSection[]` con tipo + offsets + confidence. El chunker
usa estas fronteras como puntos de corte prioritarios y asigna `sectionType`
a cada chunk.

## 3. Integración con token-chunker

```
normalizer (7.07a) → cleanText
    ↓
section-detector (7.07c) → DetectedSection[]
    ↓
token-chunker (7.07b) → Chunk[] con sectionType asignado
```

El chunker recibe `detectedSections?` como opción. Si presente:
1. Usa section boundaries como puntos de corte prioritarios (sobre sentence/legal boundaries)
2. Si una sección cabe en el rango de tokens → 1 chunk por sección
3. Si excede → subdividir manteniendo el sectionType
4. Asigna `sectionType` del DetectedSection al chunk
5. Si no hay sections → fallback a comportamiento actual ('general')
